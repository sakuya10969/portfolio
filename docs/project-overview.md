# プロジェクト概要

## サイトの目的

エンジニアとしての技術力・設計力・制作実績・学習姿勢を外部に明確に伝えるためのポートフォリオサイト。
単なるプロフィール紹介ではなく、各制作物における背景・技術選定理由・設計思想・工夫・課題・今後の改善余地まで含めて伝える構成にする。

## ターゲットユーザー

| 対象 | 想定する閲覧目的 |
|---|---|
| 採用担当者 | 技術力・実績の評価、カルチャーフィットの判断 |
| エンジニア | 技術的関心からの閲覧、設計アプローチの参考 |
| 技術に関心のある第三者 | 学生・非エンジニアの技術関係者による情報収集 |

## コアページ構成

| ページ | 役割 | 主な情報 |
|---|---|---|
| Home | ファーストビュー。人物像と代表制作物を提示し、全体への導線起点とする | 名前・肩書き・一言紹介、代表プロジェクト（2〜3件）、技術スタック概要、各セクション導線 |
| About | 経歴・価値観・技術的関心領域を伝える | 自己紹介、エンジニアとしての価値観、SNS / GitHub リンク |
| Projects | 制作物一覧。カテゴリ・技術でのフィルタリングを提供する | プロジェクトカード一覧、カテゴリフィルタ、技術フィルタ |
| Project Detail | 個別制作物の詳細。背景から改善予定まで網羅する | タイトル、概要、背景、技術スタック、設計要点、工夫、課題、改善予定、外部リンク |
| Skills / Tech Stack | 技術スタックを分類・習熟度とともに提示する | カテゴリ別技術一覧、習熟度、プロジェクトとの関連 |
| Experience | 職歴・学歴・活動歴を時系列で整理する | 期間、役割、組織名、概要 |
| Contact | 問い合わせフォーム | 名前、メール、件名、本文、バリデーション、送信フィードバック |

## 情報の優先順位

1. 制作物（Projects）— 何を作ったかが最も重要
2. 技術スタック（Skills）— 何ができるかを裏付ける
3. 経歴（Experience）— どういう文脈で活動してきたか
4. 人物像（About）— 価値観と関心領域
5. 連絡手段（Contact）— アクションへの導線

## 設計原則

- 初見で「何をやっている人間か」が伝わること
- 3クリック以内で主要プロジェクト詳細に到達できること
- 各ページは単一の責務を持ち、情報の階層は浅く保つ
- 重要な情報ほど上部・左側に配置する（F字パターン）
- 空状態・ローディング状態・エラー状態を必ず設計する
